Complete Training Script
# scripts/train_ml_model.py
import pandas as pd
import numpy as np
from sklearn.feature_extraction.text import TfidfVectorizer
from sklearn.linear_model import LogisticRegression
import pickle

# Load data
train = pd.read_csv('datasets_raw/train.csv')
train['comment_text'] = train['comment_text'].fillna("unknown")

# TF-IDF
vec = TfidfVectorizer(
    ngram_range=(1,2),
    max_features=15000,
    min_df=3,
    max_df=0.9,
    sublinear_tf=True
)
X_train = vec.fit_transform(train['comment_text'])

# Train classifiers
labels = ['toxic', 'severe_toxic', 'obscene', 'threat', 'insult', 'identity_hate']

def pr(x, y_i, y):
    p = x[y==y_i].sum(0)
    return (p+1) / ((y==y_i).sum()+1)

models = {}
for label in labels:
    y = train[label].values
    r = np.log(pr(X_train, 1, y) / pr(X_train, 0, y))
    X_nb = X_train.multiply(r)
    m = LogisticRegression(C=4, max_iter=100).fit(X_nb, y)
    models[label] = (m, r)

# Save
with open('models/ml_fast.pkl', 'wb') as f:
    pickle.dump({'vectorizer': vec, 'models': models, 'labels': labels}, f)

print("✓ Model trained and saved!")

Hybrid System: Rules + ML

Intelligent Decision Logic

The hybrid system runs both rules and ML in parallel, then intelligently combines results based on confidence levels:

# Parallel Execution Flow
┌─────────────────┐
│   Input Text    │
└────────┬────────┘
         │
         ├─────────────────┬─────────────────┐
         ↓                 ↓                 
  ┌─────────────┐   ┌─────────────┐
  │    Rules    │   │     ML      │
  │   (1.8ms)   │   │   (0.9ms)   │
  └──────┬──────┘   └──────┬──────┘
         │                 │
         └────────┬────────┘
                  ↓
         ┌─────────────────┐
         │ Decision Logic  │
         └────────┬────────┘
                  ↓
         ┌─────────────────┐
         │  Final Result   │
         └─────────────────┘

Decision Rules

Rule 1: High Confidence Rules (score > 0.8)
✅ Return immediately, don't wait for ML
Example: "I will kill you" → Rules: 1.0, return FLAG instantly
Rule 2: High Confidence ML (score > 0.8)
✅ ML caught something rules missed
Example: "you should die" → Rules: 0.0, ML: 0.96, return FLAG
Rule 3: Both Agree Safe (both < 0.3)
✅ Return SAFE with low score
Example: "have a great day" → Rules: 0.0, ML: 0.01, return SAFE
Rule 4: Disagreement or Medium Confidence
🔄 Weighted average: Rules 70%, ML 30%
Example: Rules: 0.4, ML: 0.7 → Combined: 0.49 (SAFE)
Implementation Code
# src/guards/ml_guard.py
def predict_hybrid(text: str, **kwargs) -> Dict[str, Any]:
    # Run both in parallel
    rule_result = rule_predict(text, **kwargs)
    ml_result = predict_ml(text)
    
    rule_score = rule_result['score']
    ml_score = ml_result['score']
    
    # Rule 1: High confidence rules
    if rule_score > 0.8:
        return {
            **rule_result,
            'method': 'rules_high_confidence',
            'ml_score': ml_score,
            'ml_used': True
        }
    
    # Rule 2: High confidence ML
    if ml_score > 0.8:
        return {
            'prediction': 'flag',
            'score': ml_score,
            'method': 'ml_high_confidence',
            'rule_score': rule_score,
            'ml_score': ml_score,
            'explanation': f'ML detected {ml_result["category"]}'
        }
    
    # Rule 3: Both agree safe
    if rule_score < 0.3 and ml_score < 0.3:
        return {
            'prediction': 'pass',
            'score': max(rule_score, ml_score),
            'method': 'both_agree_safe',
            'rule_score': rule_score,
            'ml_score': ml_score
        }
    
    # Rule 4: Weighted combination
    combined = (rule_score * 0.7) + (ml_score * 0.3)
    return {
        'prediction': 'flag' if combined > 0.5 else 'pass',
        'score': combined,
        'method': 'hybrid_weighted',
        'rule_score': rule_score,
        'ml_score': ml_score,
        'explanation': f'Hybrid: rules={rule_score:.2f} (70%), ml={ml_score:.2f} (30%)'
    }

Adversarial Immune System (AEGIS)

A self-evolving defense inspired by biological immune systems that learns from attacks in real-time, creates "digital antibodies" (generalized patterns), and blocks similar attacks immediately. This complements rules and ML with adaptive, data-driven defenses.

Algorithms

T-Cells (Novelty Detection)
  • • Obfuscation score: leetspeak ratio, digit substitutions, mixed case anomalies
  • • Encoded content: base64-like regex [A-Za-z0-9+/]{20,}={0,2}
  • • Unicode anomalies: non-ASCII fractions in text
  • • Entropy (Shannon): H(X) = -∑ p(x) log₂ p(x); high H ⇒ likely obfuscation
  • • Decision: anomaly_score > sensitivity ⇒ novel attack
B-Cells (Antibody Generation)
  • • Extract core pattern: generalize digits, normalize leetspeak
  • • Build regex with word boundaries to balance FP/FN
  • • Confidence initialization and fitness updates from feedback
  • • Clonal selection: keep top-fitness antibodies, mutate best
# Pseudocode (AEGIS Core)
# Detect and learn (simplified)
function detect_and_learn(text, ground_truth=None):
    # 1) Known attacks: memory cell match
    for ab in memory_cells:
        if ab.regex.match(text):
            return FLAG, ab.confidence, ab

    # 2) Novelty: T-cell heuristics
    anomaly = obfuscation_score(text) + entropy_score(text)
    if anomaly > sensitivity:
        # 3) Generate antibody (B-cell)
        pattern = extract_pattern(text)
        ab = create_antibody(pattern, confidence=0.6)
        memory_cells.add(ab)
        return FLAG, anomaly, ab

    return SAFE, 0.0, None

# CLIP-based Multi-Modal Safety Pipeline

Input Image (JPEG/PNG/WebP/GIF)
        ↓
Preprocessing (Resize to 512×512, RGB conversion)
        ↓

    • Vision Transformer Large (14×14 patches)

    • Output: 768-dimensional embedding
        ↓
Multi-Label CNN Classifier
    • Conv1D Block 1: 1 → 64 channels (768 → 384 dims)
    • Conv1D Block 2: 64 → 128 channels (384 → 192 dims)
    • Conv1D Block 3: 128 → 256 channels (192 → 96 dims)

    • FC Layers: 256 → 128 → 3 (NSFW, Violence, Hate)
        ↓
Sigmoid Activation → Probabilities [0, 1] per category
        ↓
Threshold @ 0.5 → Binary predictions (multi-label)
        ↓
Output: {prediction: "flag"|"pass", scores: {...}, latency_ms: X}

Why CLIP + CNN?

CLIP Advantages
  • • Semantic Understanding: Pre-trained on 400M image-text pairs
  • • Zero-Shot Capable: Works without training via text prompts
  • • Rich Embeddings: 768-dim captures visual + semantic features
  • • Open Source: No API costs, runs on-premise
CNN Classifier Benefits
  • • Better Feature Extraction: Convolutional layers learn spatial patterns
  • • Batch Normalization: Stable training, faster convergence
  • • Multi-Label: Images can violate multiple categories
  • • Lightweight: ~1.5M params vs 86M (ViT-L)

Zero-Shot Classification (No Training Required!)

If the trained CNN classifier is unavailable, SeaRei automatically falls back to zero-shot CLIP classification using text prompts. This requires no training data and works out-of-the-box! 

# Zero-Shot Prompts for Each Category
ZERO_SHOT_PROMPTS = {
    "nsfw": [
        "a photo of explicit sexual content",
        "a photo of nudity",
        "a pornographic image"
    ],
    "violence": [
        "a photo of graphic violence",
        "a photo of blood and gore",
        "a photo of weapons"
    ],
    "hate_symbols": [
        "a photo of a swastika",
        "a photo of KKK imagery",
        "a photo of a hate symbol"
    ],
    "safe": [
        "a photo of a landscape",
        "a photo of a person smiling",
        "a photo of everyday objects"
    ]
}

# Inference: Compute similarity between image and text embeddings
image_embedding = clip.encode_image(image)  # 768-dim
text_embeddings = clip.encode_text(prompts)  # N × 768-dim
similarities = cosine_similarity(image_embedding, text_embeddings)
prediction = argmax(similarities) if max(similarities) > threshold else "safe"
✅ Advantages
  • • No training data required
  • • Works immediately out-of-the-box
  • • Add new categories by adding text prompts
  • • Flexible and adaptable
⚠️ Limitations
  • • Lower accuracy (~70% vs 80%+ with trained classifier)
  • • May miss nuanced or context-dependent violations
  • • Requires prompt engineering for best results
  • • Less robust to adversarial examples

Python SDK
import requests

def check_content(text):
    response = requests.post(
        'http://localhost:8001/score',
        json={'text': text}
    )
    data = response.json()
    
    print(f"Text: {text}")
    print(f"Score: {data['score']:.3f}")
    print(f"Method: {data.get('method', 'unknown')}")
    print(f"Explanation: {data.get('explanation', 'N/A')}")
    
    return data['score'] > 0.5

# Examples
check_content("I will kill you")          # → True (rules caught)
check_content("you should die")           # → True (ML caught)
check_content("have a great day")         # → False (both safe)
